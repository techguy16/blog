Bring back Manifest V2 extensions to Chrome!

techguy16 July 11, 2025

These flags have been marked for deprecation since Chrome 137 and could be removed at any time. This guide may not work at some point in the near future.

In Chrome 139 and above, Manifest V2 extensions are "no longer" supported. This is not the case, as the browser still internally supports these extensions. Through some digging in the Chromium source, we (LagLife and myself) discovered that there is actually a command-line flag to re-enable these extensions. 

To re-enable Manifest V2 extensions:

Self-promo: Linux users are able to skip this guide and use LinuxPatcher to automatically modify Chrome's shortcuts to launch with the required flag.

When launching Chrome, you will have to append a command-line flag to Chrome's executable. There are two flags that work, which are for different scenarios.

Option #1
--disable-features="ExtensionManifestV2Disabled, ExtensionManifestV2Unsupported, ExtensionManifestV2DeprecationWarning"

This flag re-enables Manifest V2 extensions, hides the deprecation warning in chrome://extensions and also allows users to install Manifest V2 extensions from the Chrome Web Store. 

Option #2
--enable-features="AllowLegacyMV2Extensions"

This flag re-enables Manifest V2 extensions that were previously installed. This does not allow installing extensions from the Chrome Web Store.

How was this discovered?

This was discovered in the run_py_tests.py file in the Chromium source, which has a test that disables Google's absurd decision as many tests still rely on Manifest V2.

The extension_features.cc file has various "features" relating to disabling Manifest V2, which is where ExtensionManifestV2DeprecationWarning were discovered.
